# specs/ — 詳細仕様書フォルダ

操作者から仕様書の作成を指示された場合に、本フォルダへ格納する。

## ファイル命名規則

```
<対象処理名>.spec.md
例: ItemSortController.spec.md
    lineItemSorter.spec.md
```

## 記載テンプレート

```markdown
# <処理名> 詳細仕様書

## 基本情報

| 項目 | 内容 |
|---|---|
| 対象 | Apex クラス / LWC / Flow など |
| 作成者 | <操作者名> |
| 作成日 | YYYY.MM.DD |
| バージョン | 1.0 |

## 概要

<処理全体の目的・概要を記載>

## 引数

| 引数名 | 型 | 必須 | 説明 |
|---|---|---|---|
| <引数名> | <型> | ○/× | <説明> |

## 戻り値

| 型 | 説明 |
|---|---|
| <型> | <説明> |

## 処理詳細

### フローの場合（要素単位で記載）

| 順序 | 要素名 | 種別 | 処理内容 |
|---|---|---|---|
| 1 | <要素名> | <種別> | <処理内容> |

### プログラムの場合（コードブロック単位で記載）

#### <メソッド名 / 処理ブロック名>

- **概要**: <処理概要>
- **引数**: `<引数名>` — <型> — <説明>
- **戻り値**: `<型>` — <説明>
- **処理内容**:
  1. <ステップ1>
  2. <ステップ2>

## エラー処理

| 条件 | 対応 |
|---|---|
| <エラー条件> | <対応内容> |

## 関連資材

- <関連するクラス・LWC・Flow など>
```
