# adr/ — アーキテクチャ決定記録 (ADR) フォルダ

機能の内容に変更や追加が生じ、操作者から記録を残すよう指示された場合に、本フォルダへ格納する。

## ファイル命名規則

```
choice-<連番3桁>.md
例: choice-001.md
    choice-002.md
```

連番は作成順に採番する。欠番は作らないこと。

## 記載テンプレート

```markdown
# choice-<連番>: <決定内容のタイトル>

## 基本情報

| 項目 | 内容 |
|---|---|
| 記録者 | <操作者名> |
| 記録日 | YYYY.MM.DD |
| ステータス | 採用 / 却下 / 保留 |

## 背景・課題

<なぜこの決定が必要になったかを記載>

## 検討した選択肢

### 選択肢 A: <選択肢名>

- **概要**: <説明>
- **メリット**: <メリット>
- **デメリット**: <デメリット>

### 選択肢 B: <選択肢名>

- **概要**: <説明>
- **メリット**: <メリット>
- **デメリット**: <デメリット>

## 決定内容

**採用した選択肢**: 選択肢 <X>

<採用理由を記載>

## 影響範囲

| 資材名 | 種別 | 変更内容 |
|---|---|---|
| <資材名> | Apex / LWC / Flow | <変更内容> |

## 備考

<補足事項があれば記載>
```
